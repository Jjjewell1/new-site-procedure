Setup Environment

Get Ollama running and Cline configured to use your local model.

Navigation

Home Setup Coding GitHub Deploy Troubleshooting

1. Start Ollama

Open a terminal and run:

ollama serve

You should see output like Ollama is running. Keep this terminal open — it's your local server.

2. Confirm the Model Is Running

In another terminal, test that Ollama responds:

ollama list

You should see something like:

REPOSITORY    TAG       SIZE
ollama        llama3.2  2.8 GB

3. Open Your Project in VS Code

Navigate to your project folder and open it:

cd [YOUR_PROJECT_FOLDER]
code .

4. Configure Cline — Ollama Endpoint

Cline needs to know where your local Ollama is running. The default endpoint is:

http://localhost:11434

In VS Code, go to the Cline settings (gear icon → Settings), then set these values:

Settings Key ollama.endpoint
Value http://localhost:11434

Cline Model Name — set this to match the model you pulled, e.g., llama3.2.

5. Verify Cline Is Connected

Open a new chat in Cline and type a simple request like "say hello". If you get a response, your setup is working.

Quick Checklist

  • Ollama server is running (ollama serve)
  • Model appears in ollama list
  • Project folder open in VS Code
  • Cline endpoint set to http://localhost:11434
  • Cline returns responses when you chat
