Deploy with Coolify & Cloudflare Tunnels

Push your static site to a GitHub repo, then deploy and expose it via Coolify + Cloudflare.

Step 1: Create a New Resource in Coolify

In your Coolify dashboard, create a new resource and point it at your GitHub repo:

  • Resource Name: e.g., [YOUR_COOLIFY_PROJECT_NAME]
  • Repository URL: https://github.com/Jjjewell1/coolify-core.git
  • Branch: main

Step 2: Configure Build & Deploy Settings for a Static Site

Since this is a static site (plain HTML/CSS/JS), configure the build step to simply copy files:

Build Command:   cp -R /source/* /deploy/
Deploy Path:     /var/www/deploy
Root Directory:  . (or the project folder)

Step 3: Set Up a Cloudflare Tunnel to Expose It

Create a new tunnel in your Cloudflare Dashboard:

  1. Navigate to Cloudflare Zero Trust > Networking > Tunnels
  2. Create a new tunnel and point it at your Coolify resource URL
  3. Note the Tunnel ID (e.g., tx1234abcd) — you'll need this for DNS records

Step 4: Point Your Domain at the Tunnel

Add a CNAME record in Cloudflare DNS to map your domain to the tunnel:

Name:    newsite
Type:     CNAME
Target:   tx1234abcd (your tunnel ID)
TTL:      Auto

Step 5: Verify Everything Is Working

After the first deployment, visit:

https://newsite.jewellcore.com

You should see your site rendered correctly.

Quick Checklist

  • Create Coolify resource at [YOUR_COOLIFY_PROJECT_NAME]
  • Set build command to copy static files
  • Create Cloudflare Tunnel pointing at the Coolify URL
  • Add CNAME record: newsite → tx1234abcd
  • Visit https://newsite.jewellcore.com to verify
