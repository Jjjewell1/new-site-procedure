🎦 Deploy with Coolify & Cloudflare

Push your static site to a GitHub repo, then deploy and expose it via Coolify + Cloudflare Tunnels.

⚙ Step 1 — Create a New Project in Coolify

In your Coolify dashboard, start with a new project:

  1. Click "New Project".
  2. Name it (e.g., "Jewel Core Newsite").
  3. Inside the project, click "Add New Resource".

⚙ Step 2 — Choose a GitHub App Source for the Resource

Select your resource source:

  • Pick the GitHub App option.
  • From the list, select your repo (e.g., "coolify-core").
  • Note: This requires that you've already authorized the Coolify GitHub App on GitHub — a one-time setup if this is your first time.

⚙ Step 3 — Deploy the Resource & Check Its Port

After deploying, you need to find what port Coolify actually assigned:

  1. In your resource settings in Coolify, check the internal port.
  2. This is often NOT port 80 — it could be something else entirely.
  3. Write that number down; you'll need it for Cloudflare next.

⚠ Warning: Port Mismatch Is the #1 Mistake

The most common error in this whole process is assuming the app runs on port 80. Coolify assigns its own internal port, and it's almost always something other than 80. If your Cloudflare tunnel doesn't reach the app after setup, you likely pointed at port 80 when you should have used the actual assigned port.

⚙ Step 4 — Set Up a Cloudflare Tunnel

In your Cloudflare Dashboard:

  1. Navigate to Networking → Tunnels.
  2. If this machine doesn't have a tunnel yet, create one (it runs the cloudflared command on your host once).

⚙ Step 5 — Add Your Domain as a Route Inside the Tunnel

Inside the tunnel, add a public hostname/route:

  • Subdomain: e.g., "newsite" (so full domain becomes newsite.jewellcore.com).
  • Service URL: http://localhost:PORT — using the actual port from Step 3, not 80.

⚙ Step 6 — Done & Verify

Once saved, your domain resolves directly to the app with no manual DNS A record needed.
