Coding Best Practices

How to get the best results when working with Cline on a new project.

1. Start with a Clear Prompt

Before asking Cline to build anything, give it context:

  • What the site is about (purpose)
  • Which pages or features you need
  • Any design preferences (layout, colors, fonts)
  • Whether it's single-file or multi-page

2. Use .clinerules to Guide Cline

Every project should have a .clinerules file at the root. This tells Cline how you want it to behave — things like keeping everything in one file, using plain HTML/CSS/JS, no placeholders or unused functions.

.clinerules already exists in your project folder. Reference that file instead of rewriting the rules here.

3. Build One Feature at a Time

Don't ask Cline to build an entire site in one go unless it's very simple. Break it down:

  1. Create the homepage first
  2. Add a navigation menu
  3. Add one content section at a time
  4. Test each piece before moving on

4. Keep It Simple — No Extra Abstractions

Cline is powerful, but you don't need React or Vue for a personal reference guide. Stick with plain HTML/CSS/JS unless the project genuinely needs more.

Quick Checklist

  • Prompt includes purpose, features, and design preferences
  • .clinerules file exists in the project root
  • One feature at a time — test before moving on
  • No unnecessary frameworks or abstractions
