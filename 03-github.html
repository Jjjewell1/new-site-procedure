GitHub Workflow

Turn your local project into a GitHub repo and push it to the remote.

Your Example Repo

This walkthrough uses the following repository:

https://github.com/Jjjewell1/coolify-core.git

Step 1: Initialize a Git Repository

If you haven't already, create an empty git repo in your project folder:

git init

This creates the .git directory and starts tracking changes.

Step 2: Add Files to Track

Stage all your project files for the first commit:

git add .

This adds every tracked file. For a large initial commit, you can exclude things like build artifacts or node_modules with:

git add --exclude='node_modules' --exclude='.DS_Store'

Step 3: Make Your First Commit

Create your initial commit message — keep it descriptive but concise:

git commit -m "Initial project setup"

Step 4: Connect to the Remote Repository

Tell git where your GitHub repo is by adding a remote origin:

git remote add origin https://github.com/Jjjewell1/coolify-core.git

Step 5: Push to GitHub

Send your code to the remote for the first time:

git push -u origin main

The -u flag creates an upstream tracking reference so future pushes become simple.

Ahead — Pull Before Pushing Later

If you've made changes on a branch and want to push them, make sure your local branch is up to date with the remote first:

git fetch origin
git pull origin main

Quick Checklist

  • Run git init to create a local repo
  • Add all files with git add .
  • Commit with a clear message using git commit -m "..."
  • Connect remote origin: git remote add origin https://github.com/Jjjewell1/coolify-core.git
  • Push to GitHub: git push -u origin main
