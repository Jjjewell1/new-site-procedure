♮ GitHub Workflow

VS Code's built-in Source Control panel handles everything — no terminal needed for day-to-day work.

♮ First Time — Initialize the Repo

  1. In VS Code, click the Source Control icon in the left sidebar (the branch symbol).
  2. If no repo exists yet, you'll see an empty panel — click "Initialize Repository".

♮ Stage & Commit

Your changes will appear in the Source Control panel:

  1. Stage all changes: click the "Add All" button (or press Ctrl+Shift+A).
  2. Type a commit message in the input field at the top of the panel.
  3. Click the green checkmark icon to commit your changes.

♮ Publish Branch / Push to GitHub

This is where your site goes live on GitHub:

  1. In the Source Control panel, click the "Publish Branch" / "Publish to GitHub" button.
  2. If you're not signed in yet, VS Code will prompt you to sign into your GitHub account.
  3. You'll be asked whether to create a new repository — choose public or private.
  4. VS Code handles everything: it creates the repo on GitHub, adds the remote URL, and pushes your code in one step.

♮ Push Updates After Initial Publish

After you've published once, any future changes follow the same pattern:

  1. Stage your new or changed files (click "Add All").
  2. Type a commit message.
  3. Click the checkmark to commit.
  4. Then click "Sync Changes" or the push button in Source Control to send it up to GitHub.

★ Terminal equivalent (for reference only)

If you ever want to use the terminal instead, these commands do exactly what the VS Code UI does:

git init
git add .
git commit -m "your message"
git push
